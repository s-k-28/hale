## Editing / re-exporting

Generated by the **app-store-screenshots** editor, scaffolded at:

    ../../../hale-appstore-banners        (sibling of this repo)

It lives outside the Expo repo on purpose — its `node_modules` (React + Next)
would collide with Metro's haste map if nested inside.

```bash
cd "../../../hale-appstore-banners"   # path relative to repo root
bun install   # first time only
bun dev       # → http://localhost:3300 (3000 was busy)
```

In the editor: rewrite copy, drag/reorder/tilt screens, switch layouts, then
**Export bundle** for every Apple size (6.9"/6.5"/6.3"/6.1") as a zip. Deck state
is `hale-appstore-banners/app-store-screenshots.json`. The HALE theme
(`hale-dark`, emerald) lives in `src/lib/constants.ts`.

## Re-capturing current app screens

The screens were captured live, not mocked:

1. Build copy at `~/halecap` (the real repo path has a space, which breaks RN
   0.85's prebuilt-pod tooling; building from a space-free path fixes it).
2. `cd ~/halecap && npx expo run:ios --no-install --device "iPhone 17 Pro"`.
3. `npx convex dev` (from the repo) brings up the local backend the app needs.
4. Navigate via deep links (`hale:///today`, `/squad`, `/coach`, `/you`, `/sos`,
   `/paywall`) and `xcrun simctl io booted screenshot`.
